# Blueprint 5.2: StaffView Atomic Modularization

## Objective
Decompose the monolithic `StaffView.tsx` "God Component" (4,000+ lines) by extracting attendance-related logic and UI into dedicated feature components.

## Implementation Details
1. **Extracted Components**:
   - `src/features/staff/components/AttendanceSystem.tsx`: Central command center for clock-ins and real-time metrics.
   - `src/features/staff/components/AttendanceAnalytics.tsx`: Recharts-based performance visualization.
   - `src/features/staff/components/StaffAttendanceTable.tsx`: Workforce ledger for historical records.
2. **Logic Extraction**:
   - `src/features/staff/hooks/useAttendance.ts`: Custom hook for managing attendance state and actions (clock-in, auto-checkout).
3. **Registry Integration**:
   - Unified the type system by using the centralized `src/types/index.ts` to prevent namespace collisions.
   - Refactored `StaffView.tsx` render functions to use the new atomic components via standard prop-drilling.

## Quality Gates (B-T-G-A)
- **Build**: Successfully compiles following the `src/` migration.
- **Test**: Vitest suites `tests/unit/attendance.test.ts` pass with new component structure.
- **Grit**: Refactored in multiple passes to ensure absolute lexical scope integrity.
- **Audit**: Verified that legacy Hop-In Express paths are no longer present in imports.
