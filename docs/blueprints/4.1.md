# 📝 BLUEPRINT: Task 4.1 - System Diagnostics Panel
> **Objective**: Create a high-fidelity "Health Dashboard" component to monitor the internal stability of the ENGLABS OS.

## 🏗️ ARCHITECTURE
The `SystemDiagnostics.tsx` will be an "Organism" level component placed in the **Administration/Settings** view. It will use the `eventBus` to listen for system-wide health events.

### 🩺 SENSORS TO IMPLEMENT
1.  **Firebase Pulse**: Ping-back verification to Firestore to ensure < 250ms latency.
2.  **AI Gateway Heartbeat**: A lightweight check to `openclaw` to ensure Agent Alpha/Beta are reachable.
3.  **Regional Time Reconciliation**: Verifying that the local browser time (IST - UTC+5:30) is within +/- 5 seconds of the Firebase Server Time. (Critical for Attendance data integrity).
4.  **Memory Monitoring**: Reporting the "Heap Usage" of the browser to prevent crashes on low-end warehouse mobile devices.

## 🧪 SUCCESS CRITERIA (TDD)
- [ ] Test 1: Should report "HEALTHY" when Firestore is connected.
- [ ] Test 2: Should trigger a "SYNC WARNING" alert if local time differs from server time.
- [ ] Test 3: Components must follow the "Industrial Clean" theme (Neutral 900 background, Emerald highlights).
